# Technical Spec: AI 广告推荐信息流 Android App (Tictok Advert)

## 1. 核心技术难点与思考

### 1.1 主信息流滚动性能与多卡片混排
在 Android 信息流（Feed）应用中，卡片混排的加载流畅度是决定用户体验的核心。
- **难点**：在同一个 `LazyColumn` 中，大图卡片、小图卡片、视频播放卡片高频混用。加载大体积封面图片和在列表中初始化/销毁视频播放器容易引发主线程卡顿（Jank）。
- **思考**：通过分类型缓存优化、利用 Coil 的内存及磁盘自动双缓冲缩短加载时延。同时，对于曝光捕捉，避免使用复杂的 Viewport 状态监听逻辑，转而通过声明式的 `LaunchedEffect(ad.id)` 挂钩在卡片渲染生命周期的起点处，实现 100% 精确且对宿主滚动性能“零开销”的自动曝光埋点。

### 1.2 原生播放器在 Feed 中复用与详情页“无缝续播”
- **难点**：多卡片列表中同时播放多个视频会导致系统资源耗尽。同时，当用户从首页点击视频进入详情页时，如何做到“零闪烁、进度无缝承接”？
- **思考**：
  1. **Feed 播放器硬性单例控制**：在首页 `HomeScreen` 中维护一个唯一可变状态 `playingAdId`。当且仅当卡片 ID 与之匹配时才加载 `VideoView`，其余卡片均显示高分辨率封面 + 播放图标，确保同一时间只有一个活跃播放器，大大降低内存消耗。
  2. **状态共享与自动 Seek**：在 shared `AdvertViewModel` 中设计一个进程级内存映像 `_videoProgressMap`（ adId -> positionMs ）。当用户进入详情页时，`DetailsScreen` 从该 Map 检索保存的毫秒数并通过 `seekTo()` 跳转继续。而在离开时（`onDispose` 中），立即捕获并在 Map 中回写最新进度。该方案避开了重度 Binder 跨页面对象传递，且容错性极强。

### 1.3 远端 AI 服务不稳定性降级（Gemini 3.5 Flash Fallback）
- **难点**：当用户身处弱网或 Gemini 接口因为 API Key 缺失/超频时，AI 搜索可能直接挂起，阻断核心浏览体验。
- **思考**：设计了一个两阶段策略的 `GeminiHelper`。优先通过 HTTP 执行 Direct REST 联网解析；若超时或因各种错误失败，自动退化到本地高性能的**受控词正则匹配搜寻引擎**（Local Keywords Matcher），输出高相似度的结构化 `SearchIntent` 和友好解释性文字，整个过程对于上游 ViewModel / UI 完全透明，对用户体验无任何负面声响。

## 2. 方案对比与最终定型

### 2.1 视频播放技术方案对比

| 指标 | 方案 A：ExoPlayer (Jetpack Media3) | 方案 B：原生 VideoView + Component 状态绑定 (最终选择) |
| :--- | :--- | :--- |
| **集成开销** | 极重，需要大量底层 gradle 配置、模块兼容性和额外的渲染 Surface 绑定 | 极轻，内置于 Android SDK，完全免去库冲突和增加编译包体积 |
| **生命周期管控** | 复杂，需要高度注意释放，易发生泄漏 | 通过 Compose 的 `DisposableEffect` 自带的 `onDispose` 监听释放，完美闭航 |
| **进度对接难度** | 需要在不同 Activity / 页面间共享庞大的 ExoPlayer 实例或重构多模块依赖 | 利用全局 ProgressMap 及 `seekTo` 实现秒级续播，逻辑简单稳定 |

### 2.2 数据埋点统计方案对比

| 指标 | 方案 A：数据库级定期聚合（Group By） | 方案 B：扁平化历史事件流 + 内存实时聚合 (最终选择) |
| :--- | :--- | :--- |
| **写回性能** | 高并发下频繁查询与数据库写回竞争，影响滚动帧率 | 极佳，事件为 `Insert` 单向追加，极轻度数据库写入开销，完全非阻塞 |
| **实现灵活度** | 难以随心所欲定制近 10 条的历史监控事件流 | 完美，前台直接订阅 `analytics_events` 流，任意计算频道、样式的 CTR 对比 |

## 4. 效果评估

1. **流畅度表现**：
   在测试中，由于摒弃了主线程重构与复杂的 View 绘制切换，Feed 信息流即使大图视频频繁混排，也可保持非常顺滑的帧率。
2. **AI 自然语言命中率**：
   通过精密编写的 System Prompt 约束，Gemini 能精准将“想要在打工空余时间听考公网课的降噪耳机”解析为下述 JSON 并匹配：
   `{"channel": "电商", "tags": ["学生党", "数码推荐", "通勤", "性价比"], "explanation": "已为您检索出适合通勤、利于静心听课的高性价比降噪耳机好物。"}`
3. **数据一致性验证**：
   进入卡片 -> 自动记录曝光 -> 详情页 -> 点赞喜欢 -> 返回列表。查看 Stats 看板，曝光和所有的互动记录无错漏发生。
