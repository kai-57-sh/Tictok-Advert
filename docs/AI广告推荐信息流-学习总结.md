# AI 广告推荐信息流学习总结

## 1. 棘手问题

- 单列信息流里同时混排大图、小图、视频，状态很容易散到 UI 层。
- 离线素材更新后，旧文件和旧统计会继续被复用。
- AI 搜索如果不收敛输出格式，后续筛选就不稳定。
- 详情页和首页共用统计口径时，容易出现重复埋点。

## 2. 解决方式

- 把媒体路径、播放能力和展示判断收敛到实体层。
- 用离线包版本号控制重导入，保证素材更新可见。
- 固定 AI 输出为 `channel / tags / explanation`，并保留本地 fallback。
- 统一通过 Repository 写入统计，避免页面各自维护口径。

## 3. 踩坑复盘

- 这次项目最重要的不是“功能堆满”，而是“交付稳定、口径一致、可复现”。
- 只替换 assets 不能保证新广告生效，数据库和私有目录里的旧资源也要一起处理。
- AI 搜索如果直接输出自由文本，会让标签过滤和频道筛选变得不可控。

## 4. 后续可改进点

- 增加搜索结果缓存，减少重复请求远端模型的开销。
- 把曝光统计从“进入列表即记曝光”继续细化到更严格的可视区域口径。
- 根据可用素材继续扩充本地视频广告数量，提升视频卡片覆盖率。
- 如果项目继续演进，可把 `VideoView` 升级为 Media3，补更完整的播放控制能力。
