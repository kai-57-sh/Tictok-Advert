# 训练营课题：AI广告推荐信息流

## 题目介绍

**题目：实现单列广告信息流**

- 在内容与广告混排的推荐场景中，用户是否愿意停留，往往取决于 **信息流的呈现质量**（布局、卡片样式、加载体验）以及 **内容理解能力**（摘要、标签、搜索）。本课题要求你们实现一个 **单列广告信息流 App**：客户端实现单列信息流展示、动态化广告卡片与详情页交互，并通过云端大模型为每条广告生成摘要与智能标签，进一步提供对话式搜索体验

- 实战周期：2 周（5 月 26日 \- 6 月 7 日）

- 协作方式：单人 或者 小组制完成

- 端侧选择：iOS / Android 均可

- 难度设计：必须完成核心功能， 可选功能是加分项，完成度越高越好

**可选题目2 （开放性）**

发挥你的想象力，用AI改造抖音广告，让广告变得更有趣，用户更容易接受，需要先和培训导师沟通课题细节

### 核心功能

1. 核心交互

    1. 实现单列广告信息流布局，支持流畅滚动与列表位置保持

    2. 广告卡片支持多样式：大图、小图、视频

    3. 顶部 Tab 支持切换频道（如：精选 / 电商 / 本地 ），切换时刷新数据

    4. 点击卡片进入详情页（图文/视频详情），返回保持原位置；

    5. 支持下拉刷新、上拉加载更多

2. 资源复用：

    1. 外流 cell 的复用

    2. 播放器资源的复用

    3. 缓存池的设计

3. 数据/状态同步： 

    1. 数据获取方式（本地 mock 还是走网络请求）

    2. 数据的管理（数据的生命周期，缓存方式）

    3. 数据、状态的传递（跨页面的数据如何传递和同步），比如：详情页和信息流的点评赞状态同步

    4. 视频播放器：暂停/续播、静音/音量控制。比如： 外流视频默认暂停状态，点击按钮进行播放/进入内流自动播放

4. 动画

    1. 页面切换动画

    2. 交互的动效，比如点赞彩蛋等

5. 埋点统计

    1. 支持模拟互动：本地状态管理，点赞、收藏、分享

    2. 且支持展示「曝光/点击」等统计信息（本地模拟 \+ 可视化展示）



### 可选功能

1. 调用大模型生成每条广告摘要（用于卡片副标题或详情摘要）；生成智能标签（如：品类、风格、受众、场景），并在卡片上展示

    1. 可以使用Qwen 免费版本，部署在本地

    2. 或者实现一个简单模型服务，搭建C/S 框架

2. 支持标签点击过滤（例如只看“运动”“学生党”“性价比”）

3. 提供对话式搜索：用户用自然语言描述想看的广告，返回匹配结果

4. 图片/视频加载与缓存（首屏优先、滚动复用、失败重试）
