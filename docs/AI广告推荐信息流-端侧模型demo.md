# AI广告推荐信息流-端侧模型 Demo

## 1. 说明

当前项目没有集成真正的端侧大模型。这里的“端侧模型 demo”指的是 App 内置的本地 fallback 规则解析器，也就是 `QwenSearchHelper.parseFallbackQuery()`。

它的定位不是替代远端 Qwen3-32B，而是在无网、无密钥或接口异常时，保证搜索功能仍可演示。

## 2. 测试环境

- 时间：2026-05-28
- 执行方式：按当前源码中的关键词规则，运行两条本地查询样例

## 3. 实测结果

### Query 1

输入：

```text
我想看适合学生党的平价跑鞋广告
```

输出：

```json
{"channel":"电商","tags":["运动","学生党","性价比"],"explanation":"本地AI助手已检索到电商频道中与【运动/学生党/性价比】相关的广告宣传内容，助您快捷理解卖点。"}
```

### Query 2

输入：

```text
我想看同城好吃好玩的火锅推荐
```

输出：

```json
{"channel":"本地","tags":["本地生活","美味","数码推荐"],"explanation":"本地AI助手已检索到本地频道中与【本地生活/美味/数码推荐】相关的广告宣传内容，助您快捷理解卖点。"}
```

## 4. 结果解读

- Query 1 的输出符合预期，能正确命中 `电商 / 运动 / 学生党 / 性价比`。
- Query 2 命中了 `本地生活` 和 `美味`，但由于规则里把“推荐”映射到了 `数码推荐`，因此出现了一个误判标签。

## 5. 结论

本地 fallback 适合做端侧降级演示，但它的规则精度明显低于远端 Qwen3-32B，后续如果继续做端侧方案，应该把“关键词命中”升级为更细粒度的分类器。
