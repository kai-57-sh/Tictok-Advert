# AI广告推荐信息流-端侧模型说明

## 1. 说明

当前项目正式接入的搜索能力是云端 `Qwen3-32B`。本文件中的“端侧模型”并不指真正部署在端上的本地大模型，而是指 App 内置的本地 fallback 规则解析器，也就是 `QwenSearchHelper.parseFallbackQuery()`。

它的定位不是替代云端 `Qwen3-32B`，而是在无网、无密钥或接口异常时，保证搜索功能仍可运行和演示，作为当前项目的降级方案。

## 2. 测试环境

- 时间：2026-05-28
- 执行方式：按当前源码中的关键词规则，运行两条本地查询样例
- 当前主路径：`Qwen3-32B` 解析自然语言查询
- 当前降级路径：`parseFallbackQuery()` 做本地规则匹配

## 3. 实测结果

### Query 1

输入：

```text
我想看适合学生党的平价跑鞋广告
```

输出：

```json
{"channel":"电商","tags":["运动","学生党","性价比"],"explanation":"本地AI助手已检索到电商频道中与【运动/学生党/性价比】相关的广告宣传内容，助您快捷理解卖点。"}
```

### Query 2

输入：

```text
我想看同城好吃好玩的火锅推荐
```

输出：

```json
{"channel":"本地","tags":["本地生活","美味","数码推荐"],"explanation":"本地AI助手已检索到本地频道中与【本地生活/美味/数码推荐】相关的广告宣传内容，助您快捷理解卖点。"}
```

## 4. 结果解读

- Query 1 的输出符合预期，能正确命中 `电商 / 运动 / 学生党 / 性价比`。
- Query 2 命中了 `本地生活` 和 `美味`，但由于规则里把“推荐”映射到了 `数码推荐`，因此出现了一个误判标签。

## 5. 结论

当前项目的真实搜索主链路是云端 `Qwen3-32B`，本地 fallback 只负责异常场景下的可用性兜底。它适合用于“端侧降级能力说明”与课程验收展示，但规则精度明显低于云端模型。后续如果继续推进端侧方案，应考虑从简单关键词命中升级到更细粒度的本地分类或轻量模型方案。
