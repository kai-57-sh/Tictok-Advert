# AI广告推荐信息流-性能数据对比

## 1. 测试环境

- 时间：2026-05-28
- 数据集：`manifest.json` 中 100 条广告，图片/视频素材共 105 个文件
- 执行方式：本地 `python3` 脚本直接读取仓库内离线包并复制到临时目录

## 2. 测试方法

1. 读取并解析 `app/src/main/assets/offline_ads/manifest.json`
2. 遍历每条广告的 `coverAssetPath` 和 `videoAssetPath`
3. 把对应素材复制到临时目录，模拟首次导入到 `filesDir`

## 3. 实测结果

| 项目 | 实测结果 |
| --- | --- |
| 广告总数 | 100 |
| 素材文件总数 | 105 |
| `manifest.json` 解析耗时 | `0.27 ms` |
| 首次复制全部素材耗时 | `1768.80 ms` |

## 4. 结果解读

- 当前离线方案的主要成本集中在首次导入的文件复制。
- 解析 manifest 的时间可以忽略不计。
- 对课程演示场景而言，这个成本是可接受的，因为后续启动只需要做版本校验，不会重复复制全部资源。

## 5. 结论

离线包 + 版本比对的方案更适合当前项目：首次导入约 1.77 秒，后续启动成本低，且结果稳定可复现。
