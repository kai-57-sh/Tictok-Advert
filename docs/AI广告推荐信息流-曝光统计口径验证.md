# AI广告推荐信息流-曝光统计口径验证

## 1. 测试环境

- 时间：2026-05-28
- 设备：`Pixel_10_Pro` Android Emulator
- 包名：`com.aistudio.tictokadvert.fvxypk`
- 验证方式：`adb` 驱动真实安装包，配合 UI dump 与截图读取统计页结果

## 2. 当前口径

- 曝光：卡片进入首页列表项组合生命周期时记录一次 `view`
- 点击：从首页/搜索页触发卡片跳转时记录 `click`
- 互动：点赞、收藏、分享分别记录 `like / favorite / share`

## 3. 实测步骤与结果

### 3.1 重置统计

操作：

1. 打开统计页
2. 点击右上角“重置模拟埋点数据”

结果：

| 指标 | 实测值 |
| --- | --- |
| 累计曝光量 | 0 |
| 累计点击数 | 0 |
| 累计互动计数 | 0 |

结论：重置功能生效。

### 3.2 首页停留，不做点击

操作：

1. 从统计页返回首页
2. 停留约 3 秒，不滚动、不点击
3. 再次进入统计页

结果：

| 指标 | 实测值 |
| --- | --- |
| 累计曝光量 | 2 |
| 累计点击数 | 0 |
| 累计互动计数 | 0 |
| 频道维度 | 精选 `0 点击 / 2 曝光` |
| 卡片维度 | 小图 `0 点击 / 1 曝光`，视频 `0 点击 / 1 曝光` |

结论：当前首页首屏可见卡片会直接触发曝光，和 `LaunchedEffect(ad.id)` 的实现一致。

### 3.3 进入详情并执行一次收藏

操作：

1. 返回首页
2. 点击第一条视频广告正文进入详情
3. 返回首页后点击该广告的收藏按钮
4. 再次进入统计页

结果：

| 指标 | 实测值 |
| --- | --- |
| 累计曝光量 | 6 |
| 累计点击数 | 2 |
| 累计互动计数 | 1 |
| 频道维度 | 精选 `2 点击 / 6 曝光` |
| 卡片维度 | 视频 `2 点击 / 3 曝光` |
| 事件流顶部 | `收藏(FAV) 18支值得购入不后悔的大牌口红` |

## 4. 实测发现

- 当前安装包在“首页进入详情”时出现了点击双计数。
- 原因是首页跳转前已记录一次 `click`，详情页进入时又自动记录了一次。

## 5. 修复说明

源码已移除 `DetailsScreen` 进入时的自动 `onAdClicked(adId)`，点击事件改为只在入口页面记录一次。由于当前测试基于修复前已安装包，以上数字保留为真实抓取结果，同时用于说明问题来源。
